Projects

Unstructured to Structured Data for Corporate Invoices

Aim was to extract all intelligible information from unstructured data sans any Rule-based logic. We are working on various methods to accomplish the task. Till now, we have developed two independent pipelines for the problem statement. This project is in collaboration with a major American auto-finance company and estimated revenue generation projected is $800 per 10 invoices per month.

    Fuzzy matching(August, 2018 – October, 2018)[Psuedo Code ... Coming Soon]

    • Used Tesseract for OCR of invoices & developed the algorithm using Fuzzy matching(Levenshtein, Jaro-Winkler distance) and Regex.
    • The algorithm is independent of invoice formats.Number of entities targeted: 6.
    • Against benchmarked 5000-set of invoices (300 dpi tiff files) this method has a Precision average of 63% per entity (Precision-Recall metric)

    Coordinates(December, 2018 – Present)

    • In this pipeline, used Nuance SDK for Optical Character Recognition.
    • The invoices are read after various image processing steps like despeckling, deskew, line removals and text with their coordinates are obtained from the OCR.
    • The coordinates are leveraged to focus on in the area where the entity may be present. Cardinal numbers are used for further tunnelling in.
    • Against benchmarked 5000-set of invoices (300 dpi tiff files) this method has a Recall average of 57% per entity (Number of entities targeted: 3)
    • This method shows potential for improvement. Work to extrapolate this for more 3 entities.

Summarization

    Journal/Article Summarization (November,2018 – Present)[Code ... Coming Soon]

    Aim was to summarize any Form 10-k documents with concise & appropriate information.
    • Built a dataset from 5000 10-k forms. Section 1A and 7A were isolated from the forms, cleaned and tokenized to generate the dataset.
    • Sequence-to-sequence model using a bidirectional GRU encoder and a GRU decoder was found to give satisfactory performance.
    • Tried an architecture which is an Extractive + Abstractive model for summarization using multilayer LSTMs where the abstractive component was a seq-seq attentional model + Pointer Network + Coverage mechanism(to preempt repetitions, Abigail et.al)
      • The results were better by 5% margin.
    • Currently working on LDA based topic modeling for summarization.

Text Generation

    Conditional Text Generation (...)[Code ... Coming Soon]

    Aim was to try out Deep Generative Models for text generation.
    • Tried out GANs and VAEs for generation.
    • Tried both Policy-Gradient and Gumbel-Softmax to make GANs handle discrete outputs.
    • Used Curriculum Learning and a conditional language model to bootstrap the GANs.
    • Found both GANs and VAEs difficult to train.
    • Ultimately, a simple conditional language model was found to have the best performance.

    Question Generator for Human Welfare and Health Publisher(...)[Code ... Coming Soon]

    Able to generate all questions possible from given journal/article/paragraph.
    • A RNN encoder-decoder architecture while adopting the global attention mechanism, used for generating questions.
    • We trained our model on 37,000 question-article-parent arcticles.
    • Evaluation was based on BLEU and FM scores(92.9%). The model was able to generate 4 lucid questions from any 5 line paragraph.

    Answer Generation for Reading Comprehension (,,,)[Code ... Coming Soon]

    Aim was to generate answers to User-input questions. The answers should be generated, using content from the given database of articles/ journals
    • Used combination of Information Retrieval & Neural machine comprehension model for extractive question answering.
    • IR - First an article retriever model using bi-gram hashing and TF-IDF weighted bag-of-word vectors which if given a question, efficiently returns a subset of relevant articles.
    • Now these articles passed to a multi-layer recurrent neural network model which was trained to detect answer spans in them.
    • We used combined corpus of Wikipedia articles, SQuAD & propriety journals (SQLite3)
    • Dataset Size : 15GB, F1 score: 96%

    Customer Service Auto-Reply System(...)[Code ... Coming Soon]

    Key problem was to Auto-generate responses on the Customer service side, where the response corresponds to only one query category, from available 14 query classes (Order modification, cancellation etc.)
    • Initially had to create query & response dataset from 3000 email threads. Then, extensive pre-processing steps(language detection, tokenization, sentence segmentation) on them.
    • First, classification of user queries using SVM into pre-decided 14 query classes. This helped to identify class imbalance.
    • Tried SMOTE analysis for balancing classes but not good results for we had to lose some data in the process. Hence, we manually oversampled and under-sampled classes.
    • Then, used an encoder-decoder architecture for generating the response. LSTMs used for response selection. Response generation included semantic Intent clustering and cluster validation.

Fault/Anomaly detection

    Fault/Anomaly detection among printing logos on packages

    Aim was to predict 3 minutes into future when a logo printer will deviate from printing accurate logo design, using Machine Learning.
    • Crunched Terabytes of data to analyze coverage and signal from every one of 1500 sensor. Cleaned & segregated out 105 sensor data which were with useful statistics and signal reads (minimal noise, pattern)
    • 4 main parameters extracted out of provided data: Temperature, Tension, Pressure, Speed.
    • Further analysis of the 105 sensor signal data to find time based features which are further transformed into statistics based features over several time windows.
    • Developed a system to predict three minutes into future using LSTMs
    • Precision range of 85-90% achivied at different units.
